## 2. 问题背景

### 中职数学教学三大痛点

#### 痛点1：学习动力不足
- **数据**：调查显示60%的中职学生认为"学数学没用"
- **原因**：数学知识与实际生活、职业场景缺乏连接
- **表现**：学生"不愿意学"

#### 痛点2：教学方式单一
- **现状**：传统讲授式教学为主，缺乏个性化辅导
- **问题**：一个大班50-60人，教师难以兼顾每个学生
- **表现**：学生"学不会"

#### 痛点3：专业脱节严重
- **现状**：数学知识与职业技能缺乏联系
- **问题**：学生不知道学了数学能做什么
- **表现**：学生"不知道有什么用"

### 典型场景数据
- 场景1：汽修专业学生问"学三角函数有什么用？"（占比15%）
- 场景2：电商专业学生问"学函数对销售有什么帮助？"（占比12%）
- 场景3：数控专业学生在编程时需要用到不等式（占比10%）
- 场景4：会计专业学生在计算贷款还款时需要用到数列（占比8%）
- 场景5：建筑专业学生在计算屋顶高度时需要用到三角函数（占比7%）

### 学生需求调查数据
- 需求1：希望知道"学这个有什么用"（68%）
- 需求2：希望有人能随时解答问题（55%）
- 需求3：希望题目能结合自己专业（52%）
- 需求4：希望教学方法更生动有趣（48%）
- 需求5：希望能看到实际应用例子（45%）

## 3. 解决方案

### 核心理念
**让中职学生"愿意学数学、能学会数学、知道数学有什么用"**

### 技术架构

#### 大模型
- **产品**：豆包（Doubao）
- **模型**：doubao-seed-1-8-251228
- **特点**：多模态Agent优化模型，支持图片识别、长上下文推理
- **选择理由**：国产大模型、性价比高、支持视觉理解

#### 开发框架
- **框架**：LangChain 1.0 + LangGraph
- **功能**：Agent编排、状态管理、工具集成
- **特点**：支持有状态、可复现的Agent工作流

#### 智能体平台
- **平台**：Coze智能体搭建平台
- **功能**：低代码开发、Web SDK部署
- **特点**：快速部署、多端访问

#### 编程语言
- **语言**：Python 3.12
- **特点**：AI领域主流语言、生态完善

#### 依赖管理
- **工具**：uv
- **特点**：快速、可靠的Python包管理器

### 核心能力

#### 能力1：苏格拉底式提问引导
- 识别学生困难点
- 分步提问，避免信息轰炸
- 每次最多3个问题
- 不直接给答案

#### 能力2：专业场景自动匹配
- 28个专业场景库
- 6个专业方向
- 7个数学知识点
- 自动关联职业应用

#### 能力3：双模式支持
- 学生模式：教学辅导、学习引导
- 教师模式：分层出题、备课辅助

#### 能力4：多模态输入
- 支持文字输入
- 支持图片识别（拍照传题）

#### 能力5：短期记忆
- 保留最近20轮对话
- 上下文连贯

#### Step 4：Agent代码实现（借助豆包大模型）

**操作步骤**：
1. 向豆包大模型描述Agent需求
2. 获取LangChain框架代码
3. 整合配置和工具

**核心代码实现**：

```python
# agent.py 完整代码
import os
import json
from typing import Annotated
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langgraph.graph import MessagesState
from langgraph.graph.message import add_messages
from langchain_core.messages import AnyMessage
from coze_coding_utils.runtime_ctx.context import default_headers
from storage.memory.memory_saver import get_memory_saver

LLM_CONFIG = "config/agent_llm_config.json"

# 默认保留最近 20 轮对话 (40 条消息)
MAX_MESSAGES = 40

def _windowed_messages(old, new):
    """滑动窗口: 只保留最近 MAX_MESSAGES 条消息"""
    return add_messages(old, new)[-MAX_MESSAGES:]

class AgentState(MessagesState):
    messages: Annotated[list[AnyMessage], _windowed_messages]

def build_agent(ctx=None):
    workspace_path = os.getenv("COZE_WORKSPACE_PATH", "/workspace/projects")
    config_path = os.path.join(workspace_path, LLM_CONFIG)

    with open(config_path, 'r', encoding='utf-8') as f:
        cfg = json.load(f)

    api_key = os.getenv("COZE_WORKLOAD_IDENTITY_API_KEY")
    base_url = os.getenv("COZE_INTEGRATION_MODEL_BASE_URL")

    llm = ChatOpenAI(
        model=cfg['config'].get("model"),
        api_key=api_key,
        base_url=base_url,
        temperature=cfg['config'].get('temperature', 0.7),
        streaming=True,
        timeout=cfg['config'].get('timeout', 600),
        extra_body={
            "thinking": {
                "type": cfg['config'].get('thinking', 'disabled')
            }
        },
        default_headers=default_headers(ctx) if ctx else {}
    )

    return create_agent(
        model=llm,
        system_prompt=cfg.get("sp"),
        tools=[],
        checkpointer=get_memory_saver(),
        state_schema=AgentState,
    )
```

### 典型案例

#### 案例1：汽修专业三角函数学习（15轮对话）

**学生**：我听不懂sin、cos、tan是什么

**数小伴**：同学别着急！咱们完全不用记那些字母，先把它们当成三个"直角三角形专属计算器"，用汽修场景给你讲明白：

**数小伴**：先搞懂一个前提：咱们只看直角三角形（就是有一个90°角的三角形，汽修里测零件、调定位全是这种三角形），随便挑一个锐角，给它起个名字叫「角A」：
- 角A对面的那条边 → 叫「对边」
- 角A旁边的那条直角边 → 叫「邻边」
- 最长的那条边 → 叫「斜边」

**学生**：哦，那这三个字母是什么意思？

**数小伴**：三个"计算器"的功能：
1. **tan（正切）= 对边 ÷ 邻边** → 专门算「角度和坡度」
   比如汽修里调汽车前轮前束，垫块的垂直高度（对边）是5cm，水平长度（邻边）是12cm，tanA=5÷12≈0.42，查一下表就知道这个角度大概是22°，这就是前束角的参考值。

2. **sin（正弦）= 对边 ÷ 斜边** → 专门算「高度和长度的关系」
   比如换避震弹簧，弹簧实际长度（斜边）是100mm，倾斜角度是30°，那弹簧翘起来的高度（对边）= 100×sin30°=100×0.5=50mm，这就是你要预留的安装空间。

3. **cos（余弦）= 邻边 ÷ 斜边** → 专门算「水平距离和长度的关系」
   还是刚才的弹簧，水平方向占用的空间（邻边）=100×cos30°≈100×0.87=87mm，这样就知道底盘能不能放下这个弹簧了。

**学生**：哦，我大概懂了，那对边找sin，邻边找cos？

**数小伴**：对的！一句话记牢：**对边找sin，邻边找cos，坡度找tan**

现在来个小测试：汽修里测一个直角三角形垫块，斜边是10cm，对边是6cm，那这个角的sin值是多少呀？

**学生**：是6除以10等于0.6？

**数小伴**：太棒了！🎉 你完全理解了！sin就是对边÷斜边，你记住了公式，也计算正确了。

现在能不能试着想一下，如果你调避震时知道角度是30度，弹簧长度100mm，怎么算水平占用的空间呀？

**学生**：用cos对吧？100×cos30度，大概是100×0.87=87mm？

**数小伴**：非常好！👏 你已经掌握了三角函数在汽修里的实际应用！以后工作中遇到角度计算，记住：
- 想算高度 → 用sin
- 想算水平距离 → 用cos
- 想算角度或坡度 → 用tan

要不要来一道类似的题巩固一下？数值我换简单的。比如一个三角形斜边20cm，对边12cm，你试试算sin、cos和角度？

**学生**：sin=12÷20=0.6，cos=...这个不知道邻边

**数小伴**：没关系！这是常见的情况。我们先算邻边，用勾股定理：邻边² + 对边² = 斜边²
所以邻边² = 20² - 12² = 400 - 144 = 256
邻边 = √256 = 16cm

现在可以算cos了：cos = 邻边÷斜边 = 16÷20 = 0.8

角度呢？tan = 对边÷邻边 = 12÷16 = 0.75，查表可知角度大概是37°

怎么样，这个思路清楚了吗？

**学生**：清楚了！谢谢数小伴！

**数小伴**：不客气！记住，汽修里很多测量和计算都用到三角函数，以后遇到实际问题，想想这三种"计算器"怎么用就行。加油！💪

- 基础题中，学生可能会在等差数列的项数计算上出错，比如第一题求第五周的销量，容易错误地认为是100 + 20×5，而忽略了第一周已经是初始量，实际是100 + 20×(5 - 1)。

- 挑战题中，对于递推公式的处理容易出现混乱，比如第二题，学生可能难以将bn=bn-1+200-50n转化为求最值的形式。

- 讲挑战题第二题时，提问："我们怎么把这个递推公式bn=bn-1+200-50n转化为我们熟悉的可以求最值的形式呢？比如能不能先求出bn的通项公式？"

### 示例2：概念讲解

**用户**：帮我解释一下什么是二次函数

**数小伴**：没问题！咱们先从生活里的例子入手，把二次函数讲明白：

你有没有见过扔篮球？篮球飞出去的轨迹就是一条开口向下的曲线，这个轨迹的高度和时间的关系，就是二次函数。简单说，**二次函数就是一个"有最高点或最低点"的函数，它的表达式里一定有x的平方项（比如x²），而且x²的系数不能是0**。

### 给你拆成3个好懂的点：
1. **核心特征**：表达式长得像 y = ax² + bx + c（a、b、c是固定的数，a≠0）。比如 y = x²、y = 2x² + 3x - 1 都是二次函数，没有x²的就不是哦。
2. **图像样子**：是一条叫"抛物线"的曲线，要么开口向上（像碗），有最低点；要么开口向下（像倒扣的碗），有最高点。比如刚才说的篮球轨迹，就是开口向下的抛物线，最高点就是篮球能飞到的最大高度。
3. **现实用处（结合专业）**：
    - 如果你学电商/营销：计算商品的利润时，利润和定价的关系就是二次函数——定价太低赚得少，定价太高没人买也赚得少，中间有个"利润最大化"的定价，就是这个二次函数的最高点。
    - 如果你学数控/建筑：加工弧形零件、计算拱形门的高度时，也会用到二次函数的曲线规律。

### 小测试帮你巩固：

B. y = x² - 5
C. y = 5

#### 挑战题（对应汽修改装场景）
1. 改装汽车的进气歧管时，需要设计一个倾斜的连接管：已知连接管一端的高度比另一端高15cm，连接管的水平投影长度为40cm，要让连接管与水平方向的夹角满足tanθ=0.4，需要将另一端的高度调整多少厘米？
2. 汽车刹车盘的某一磨损面是一个斜面，已知磨损面的长度为25cm，磨损面与刹车盘平面的夹角为20°，求磨损面的垂直磨损深度（结果保留1位小数，参考值：sin20°≈0.34，cos20°≈0.94，tan20°≈0.36）。

### 教学辅助资料

#### 常见错误预判
1. 基础题中容易混淆直角三角形的"对边/邻边"，比如计算tanα时把水平长度当对边；
2. 应用题中忽略角度的实际意义，比如把外倾角当成与垂直方向的夹角；
3. 挑战题中不会将实际问题转化为三角函数的等式，比如调整高度时搞不清是加还是减。

3. 讲挑战题时问："如果tanθ的要求变了，我们怎么快速计算需要调整的高度？"（引导学生总结三角函数的应用规律）

## 12. 代码片段

### agent.py完整代码

```python
import os
import json
from typing import Annotated
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langgraph.graph import MessagesState
from langgraph.graph.message import add_messages
from langchain_core.messages import AnyMessage
from coze_coding_utils.runtime_ctx.context import default_headers
from storage.memory.memory_saver import get_memory_saver

LLM_CONFIG = "config/agent_llm_config.json"

# 默认保留最近 20 轮对话 (40 条消息)
MAX_MESSAGES = 40

def _windowed_messages(old, new):
    """滑动窗口: 只保留最近 MAX_MESSAGES 条消息"""
    return add_messages(old, new)[-MAX_MESSAGES:]

class AgentState(MessagesState):
    messages: Annotated[list[AnyMessage], _windowed_messages]

def build_agent(ctx=None):
    workspace_path = os.getenv("COZE_WORKSPACE_PATH", "/workspace/projects")
    config_path = os.path.join(workspace_path, LLM_CONFIG)

    with open(config_path, 'r', encoding='utf-8') as f:
        cfg = json.load(f)

    api_key = os.getenv("COZE_WORKLOAD_IDENTITY_API_KEY")
    base_url = os.getenv("COZE_INTEGRATION_MODEL_BASE_URL")

    llm = ChatOpenAI(
        model=cfg['config'].get("model"),
        api_key=api_key,
        base_url=base_url,
        temperature=cfg['config'].get('temperature', 0.7),
        streaming=True,
        timeout=cfg['config'].get('timeout', 600),
        extra_body={
            "thinking": {
                "type": cfg['config'].get('thinking', 'disabled')
            }
        },
        default_headers=default_headers(ctx) if ctx else {}
    )

    return create_agent(
        model=llm,
        system_prompt=cfg.get("sp"),
        tools=[],
        checkpointer=get_memory_saver(),
        state_schema=AgentState,
    )
```
