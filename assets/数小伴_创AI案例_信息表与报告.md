# 开发与应用报告

## 一、开发背景

### 1. 需求分析

**教学痛点**
中等职业教育数学教学面临三大核心问题：

1. **学习动力不足**：中职学生对数学普遍存在畏难情绪，认为数学抽象难懂、与自己未来职业无关。调研显示，超过60%的中职学生表示"不知道学数学有什么用"。

2. **教学方式单一**：传统课堂以讲授为主，缺乏个性化辅导。教师难以关注每个学生的具体问题，学生遇到困难时得不到及时有效的帮助。

3. **专业脱节**：数学知识与专业技能教学相互割裂，学生无法理解数学在实际工作中的应用价值。如三角函数与汽修专业的前束角计算、二次函数与电商的利润分析等。

**应用场景**
本智能体主要应用于中等职业学校数学教学的以下场景：
- 课后个性化辅导：学生可随时提问，获得针对性解答
- 课前预习引导：通过生活化和专业化的例子激发学习兴趣
- 教师备课辅助：快速生成分层作业题和教学建议
- 课后巩固练习：根据学生掌握情况推送类似题目

### 2. 市场调研

**现有解决方案分析**
目前市场上已有一些数学教育智能产品，但普遍存在以下问题：

1. **面向普通教育**：大多数产品针对初高中或大学，缺乏对中职学生的理解，题目和专业场景不匹配。

2. **直接给答案**：许多产品采用"输入题目→输出答案"的模式，不利于学生思维能力培养。

3. **交互机械化**：缺乏情感关怀和激励，难以激发中职学生的学习兴趣。

**技术可行性**
随着国产大模型的发展，基于生成式AI的教育智能体开发已具备技术基础：
- 豆包、DeepSeek、智谱等大模型在中文理解和对话能力上表现优异
- Coze等智能体搭建平台提供了低代码开发环境
- LangChain等框架简化了Agent开发流程

**核心代码实现**：

**文件：src/agents/agent.py**
```python
import os
import json
from typing import Annotated
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI
from langgraph.graph import MessagesState
from langgraph.graph.message import add_messages
from langchain_core.messages import AnyMessage
from coze_coding_utils.runtime_ctx.context import default_headers
from storage.memory.memory_saver import get_memory_saver

# 配置文件路径
LLM_CONFIG = "config/agent_llm_config.json"

# 记忆窗口：保留最近20轮对话（40条消息）
MAX_MESSAGES = 40

def _windowed_messages(old, new):
    """滑动窗口：只保留最近MAX_MESSAGES条消息"""
    return add_messages(old, new)[-MAX_MESSAGES:]

class AgentState(MessagesState):
    messages: Annotated[list[AnyMessage], _windowed_messages]

def build_agent(ctx=None):
    """构建数小伴Agent"""
    workspace_path = os.getenv("COZE_WORKSPACE_PATH", "/workspace/projects")
    config_path = os.path.join(workspace_path, LLM_CONFIG)

    # 读取配置文件
    with open(config_path, 'r', encoding='utf-8') as f:
        cfg = json.load(f)

    # 配置大模型
    api_key = os.getenv("COZE_WORKLOAD_IDENTITY_API_KEY")
    base_url = os.getenv("COZE_INTEGRATION_MODEL_BASE_URL")

    llm = ChatOpenAI(
        model=cfg['config'].get("model"),  # doubao-seed-1-8-251228
        api_key=api_key,
        base_url=base_url,
        temperature=cfg['config'].get('temperature', 0.7),
        streaming=True,
        timeout=cfg['config'].get('timeout', 600),
        extra_body={
            "thinking": {
                "type": cfg['config'].get('thinking', 'disabled')
            }
        },
        default_headers=default_headers(ctx) if ctx else {}
    )

    # 创建Agent
    return create_agent(
        model=llm,
        system_prompt=cfg.get("sp"),  # 系统提示词
        tools=[],  # 暂不使用额外工具
        checkpointer=get_memory_saver(),  # 短期记忆
        state_schema=AgentState,  # 状态管理
    )
```

**测试对话示例**（场景：三角函数）：
```
学生：我听不懂sin、cos、tan是什么

数小伴：同学别着急！咱们完全不用记那些字母，先把它们当成三个"直角三角形专属计算器"，用汽修场景给你讲明白：

先搞懂一个前提：咱们只看直角三角形，随便挑一个锐角，给它起个名字叫「角A」：
- 角A对面的那条边 → 叫「对边」（比如你调避震时，弹簧翘起来的高度）
- 角A旁边的那条直角边 → 叫「邻边」（比如弹簧的水平距离）
- 最长的那条边 → 叫「斜边」（比如弹簧的实际长度）

三个"计算器"的功能：
1. tan = 对边÷邻边 → 专门算角度和坡度
2. sin = 对边÷斜边 → 专门算高度和长度的关系
3. cos = 邻边÷斜边 → 专门算水平距离和长度的关系

一句话记牢：对边找sin，邻边找cos，坡度找tan

现在来个小测试：汽修里测一个直角三角形垫块，斜边是10cm，对边是6cm，那这个角的sin值是多少呀？

学生：0.6

数小伴：你这一步代数值进去算对了，很多人会在这里忘掉负号。要不要来一道类似的题巩固一下？数值我换简单的。
```

### （三）功能架构

数小伴智能体包含4大功能模块、8个核心子系统，形成完整的闭环教学体系。

#### 1. 对话引导系统
负责识别用户意图，提供个性化的教学引导。

**子系统1：场景识别模块**
- **功能**：自动识别学生输入的类型（题目/概念/应用/困难/教师）
- **实现方式**：基于系统提示词的意图分类
- **输入**：用户文本或图片
- **输出**：场景标签（如"场景A：学生问一道具体题目"）

**子系统2：问题诊断模块**
- **功能**：识别学生具体困难点
- **诊断维度**：
  - 概念不清：对基本定义不理解
  - 公式记错：记错公式或参数
  - 计算错误：计算过程出错
  - 完全没思路：不知道从何入手
- **实现方式**：苏格拉底式提问

**子系统3：分步提问模块**
- **功能**：引导学生逐步思考
- **约束**：每次最多3个问题
- **实现方式**：系统提示词控制 + 状态管理

#### 2. 专业场景库系统
将数学知识与专业应用紧密结合。

**子系统4：场景匹配模块**
- **功能**：根据知识点自动推荐专业场景
- **覆盖范围**：
  - 7个数学知识点（函数、二次函数、数列、三角函数、统计、概率、几何）
  - 6个专业方向（汽修、电商、数控、会计、建筑、物流）
  - 28个具体场景
- **实现方式**：系统提示词中的场景规则

**数据结构**：
```
知识点 → 专业 → 场景描述 → 实际应用案例 → 计算示例
```

**示例**：
```
三角函数 → 汽修 → 前束角调整
  实际应用：调整汽车前轮前束角
  计算示例：已知垫块高5cm、长12cm，求角度
  公式：tanα = 5/12
```

#### 3. 教师辅助系统
为教师提供备课和教学支持。

**子系统5：分层出题模块**
- **功能**：按难度生成分层作业
- **分层结构**：
  - 基础题：巩固基本概念和公式（2题）
  - 应用题：结合实际问题应用（2题）
  - 挑战题：综合运用和拓展（2题）
- **定制化**：支持按专业定制

**子系统6：错误预判模块**
- **功能**：预测学生常见错误类型
- **内容**：错误类型 + 产生原因 + 规避方法

**子系统7：提问建议模块**
- **功能**：提供课堂提问引导
- **内容**：提问话术 + 预期回答 + 延伸问题

#### 4. 激励反馈系统
提供情感支持和学习激励。

**子系统8：互动反馈模块**
- **功能1：具体表扬**
  - 针对性肯定学生的正确步骤
  - 示例："你这一步代数值进去算对了，很多人会在这里忘掉负号。"

- **功能2：友好纠错**
  - 不打叉，换角度引导
  - 示例："这个想法很常见，我们换一个角度再看一下……"

- **功能3：巩固建议**
  - 每完成一个小环节，询问是否需要类似题练习
  - 示例："要不要来一道类似的题巩固一下？数值我换简单的。"

## 四、创新与反思

### 创新点

1. **交互模式创新**
   - 采用苏格拉底式提问，区别于传统"问答式"智能体
   - 三问题限制机制，避免信息轰炸，降低学生认知负担
   - 情感激励系统，包含具体表扬和友好纠错，增强学习信心

2. **场景挖掘创新**
   - 系统性构建中职数学专业场景库，覆盖6个专业、7个知识点、28个应用场景
   - 将抽象数学知识与具体职业任务紧密结合，解决"学数学有什么用"的核心痛点

3. **成本控制创新**
   - 采用Coze低代码平台，降低开发门槛和维护成本
   - 使用豆包大模型（国产，性价比高），控制API调用成本
   - 短期记忆机制（20轮对话），平衡用户体验与成本

4. **双模式设计**
   - 学生模式：对话引导、专业连接、情感激励
   - 教师模式：分层出题、错误预判、提问建议
   - 一套系统，服务双重用户

### 遇到的问题及改进思路

**问题1：Web SDK访问不稳定**
- **现象**：学生通过Coze Web SDK访问时，偶尔出现消息无法发送或长时间加载
- **原因**：网络波动或并发请求导致的连接超时
- **改进思路**：
  - 增加请求重试机制
  - 优化流式响应的稳定性
  - 考虑添加离线缓存功能

**问题2：部分专业场景覆盖不够**
- **现象**：目前主要覆盖汽修、电商、数控等专业，缺少旅游、护理等专业的场景
- **原因**：开发时优先选择了工科类专业
- **改进思路**：
  - 扩充专业场景库，增加服务类专业的数学应用案例
  - 提供自定义场景功能，允许教师上传自己的专业案例
  - 建立场景共享社区，鼓励教师贡献优质案例

**问题3：缺乏学习进度追踪**
- **现象**：学生多次使用后，系统无法了解其学习进度和薄弱环节
- **原因**：当前仅实现了短期记忆（20轮对话），无长期存储
- **改进思路**：
  - 增加用户注册和登录功能
  - 建立学习档案，记录知识点掌握情况
  - 根据学习进度推送针对性练习

**未来发展方向**

1. **多模态升级**：支持学生上传手写题目的照片识别
2. **协作功能**：支持教师创建班级，共享智能体和教学资源
3. **数据分析**：提供学情分析报告，帮助教师了解整体学习情况
4. **开源分享**：将系统代码和场景库开源，促进教育智能体生态发展
